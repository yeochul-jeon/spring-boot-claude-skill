# 회고 #16 — spring-persistence empirical 검증 (결함 K: Service Entity 반환 항목 누락)

날짜: 2026-04-22

---

## 사건 요약

`spring-persistence` SKILL.md 의 구조는 완성 상태였으나(2026-04-21 작성)
실제 코드 생성 후 체크리스트 적용(empirical 검증)은 수행되지 않았다.

서브에이전트로 "JPA 기반 주문(Order) CRUD REST API (OrderItem 1:N 포함)" 시나리오를 생성하고
`spring-persistence` SKILL.md 를 참조하지 않은 맨세션 조건에서 결함을 수집했다.
5개 결함이 식별되었으며 그 중 하나(결함 K)는 SKILL.md 체크리스트에 항목이 없었다.

---

## 발견된 결함

### 결함 K — Service Entity 반환 체크리스트 항목 누락

- 기존 SKILL.md: "Entity가 Controller 응답 타입으로 직접 노출되지 않음" — 응답 타입만 확인
- 실제 코드: `OrderService.createOrder()`, `getOrder()`, `updateOrderStatus()`, `cancelOrder()` 가 모두 `Order` Entity 반환
  → Controller 가 `OrderResponse.from(order)` 로 매핑하여 응답 타입은 DTO 이지만,
    Service ↔ Controller 경계에서 Entity가 그대로 흐름
- `spring-principles/templates/dto-entity-separation.md` After 패턴에 "Service returns DTO"가 암묵적으로 명시되어 있으나 spring-persistence 체크리스트에는 전파되지 않은 상태
- 결론: 기존 원칙의 **enforcement gap** (신규 원칙이 아닌 persistence 체크리스트 누락)

### 결함 P7 — TestContainers 의존성 누락 (탐지 공백)

- SKILL.md 체크리스트 항목 존재: "`testcontainers:mysql` (또는 해당 DB) 의존성 포함"
- 생성된 `build.gradle.kts`: `testcontainers` 의존성 없음 (`spring-boot-starter-test` 만 포함)
- grep 자동 검증 패턴 부재 → 체크리스트 항목이 있어도 빠르게 탐지할 방법 없음

### spring-web 교차 결함 (W2·W3·W5) — 기존 spring-web 체크리스트 커버

| 결함 | 내용 |
|------|------|
| W2 | `@RestControllerAdvice` 없음 |
| W3 | Controller 내부 `@ExceptionHandler` (2개) |
| W5 | `@RequestBody`에 `@Valid` 미적용 |

> 세 결함 모두 spring-web SKILL.md 체크리스트가 커버하지만,
> spring-persistence SKILL.md 는 `spring-principles` 체크리스트 실행만 명시하고
> `spring-web` 체크리스트 실행을 명시하지 않아 탐지 누락.

## 교훈

1. **"Controller 응답 타입" 체크 ≠ "Service 반환 타입" 체크.**
   Controller 가 DTO 를 응답해도 Service 가 Entity 를 반환하는 구조는 계층 경계를 오염시킨다.
   `dto-entity-separation.md` After 패턴이 이미 "Service returns DTO"를 보여주지만
   persistence 체크리스트에 항목이 없으면 탐지되지 않는다.

2. **spring-persistence 시나리오에서도 spring-web 결함이 3건 발생했다.**
   persistence 에 집중한 맨세션은 예외 처리·Validation 을 자연스럽게 누락한다.
   spring-persistence SKILL 이 `spring-web` 체크리스트 실행을 명시해야
   교차 결함을 체계적으로 탐지할 수 있다.

3. **grep 패턴 없는 체크리스트 항목은 눈 리뷰에 의존한다.**
   결함 P7(TestContainers 누락)은 항목이 있었으나 grep 자동화가 없어
   수동 확인이 필요했다. grep 섹션 추가로 이후 세션은 커맨드 한 줄로 탐지 가능.
