## 발견된 결함

### 결함 J — Request DTO 미사용 체크리스트 항목 누락

- 기존 SKILL.md: `@RequestBody` 파라미터에 `@Valid` 적용 여부만 확인
- 실제 코드: `@RequestBody Map<String, String>` 으로 raw 타입 직접 수신
  → `MemberRegisterRequest` DTO 가 dto/ 에 존재하지만 Controller 에서 미사용
- `@Valid`는 형식상 추가할 수 없어 체크리스트 항목을 통과하는 것처럼 보임
- SKILL.md 어디에도 "DTO record 를 사용해야 한다"는 항목이 없었음

### 결함 W1~W6 (기존 항목, grep 패턴 부재)

| 결함 | 내용 |
|------|------|
| W1 | Controller 가 `Member` Entity 를 직접 반환 (3개 메서드) |
| W2 | `@RestControllerAdvice` 없음 |

## 교훈

1. **`@Valid` 체크 ≠ DTO 사용 체크.**
   `@RequestBody Map<String, String>` 에는 `@Valid`를 추가할 수 없어
   체크리스트가 조용히 통과된다. "DTO record 를 써야 한다"는 항목이
   `@Valid` 항목과 별개로 필요하다.

2. **grep 패턴 7개가 없으면 7번의 눈 리뷰가 필요하다.**
   spring-web 의 모든 결함(W1~W6, J)은 grep 으로 즉시 탐지 가능하다.
   SKILL.md 에 grep 섹션을 고정하면 이후 세션이 "체크리스트 실행"을
   grep 커맨드로 구체화할 수 있다.

3. **spring-web 전용 테스트 케이스(케이스 6)가 없었다.**
   기존 케이스 1~5 중 spring-web 표면(ProblemDetail, @RestControllerAdvice,
   URL 규칙)을 직접 검증하는 케이스가 없어 리그레션 훅이 부재했다.
   케이스 6 추가로 보완.
