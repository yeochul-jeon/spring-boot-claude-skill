## 원인 분석 — 결함 A~F

**A. `.RELEASE` 접미사로 Gradle 플러그인 해석 실패**
- `start.spring.io` 메타데이터가 `"bootVersion": "4.0.5.RELEASE"` 반환
- Gradle 플러그인 포털은 `X.Y.Z` 형식만 지원; `.RELEASE` 포함 시 resolution 실패
- `fetch-latest-versions.sh` 에 정제 로직 부재

**B. `generate-project.sh` macOS tr 비호환 (기수정)**
- `tr -d '-_'` → macOS BSD tr "illegal option -- _"
- 테스트 도중 로컬 패치 `sed 's/[-_]//g'` 로 수정됨 (상류 반영 완료 확인)

**C. `apply-package-structure.sh` 상대 경로 실행 실패**
- 테스트 디렉터리 CWD 에서 `.claude/skills/...` 상대 경로 호출 실패
- SKILL.md §4 에 호출 패턴 미명시

**D. Gradle 로컬 버전 7.5.1 — Boot 4.x 요구치 8.x 미달**
- `gradle wrapper` 명령 자체가 build.gradle.kts 플러그인 해석에 실패
- SKILL.md 에 wrapper 생성 단계·최소 버전 지침 부재
- `gradle-conventions.md` 에 호환표 없음

**E. Claude Code Bash `cd` CWD 비영속**
- `cd order-api && ...` 후속 Write 가 `order-api/order-api/...` 에 중첩 생성
- SKILL.md 에 CWD 비영속 경고 없음

**F. 중단 프로토콜 후 "사용자 구두 버전" 우회로 허용**
- Claude 가 중단 선언 후 "직접 버전을 알려주시면 2-c fallback 으로 진행 가능" 제시

## 교훈

- 스크립트 출력값이 다운스트림 도구(Gradle 플러그인 포털) 형식과 맞는지
  **경계 지점에서 정규화**해야 한다. API 반환값을 그대로 전달하면 포맷 불일치가 빌드 실패로 이어진다.
- "중단" 규칙도 **허용된 우회로를 닫아야** 의미가 있다. "학습 데이터 금지" 만 적으면
  "사용자 제공 값"이라는 맹점이 남는다.
- Claude Code 의 Bash CWD 비영속은 모든 파일 생성 플로우에서 잠재적 지뢰다.
  스킬 문서 §0 에 명시적 경고가 필요하다.
