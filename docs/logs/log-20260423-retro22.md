## 신규 스킬 핵심 규칙 요약

### spring-batch

- `SimpleAsyncTaskExecutor` 사용 금지 (배치 Job 내 비동기 처리 시)
- JobRepository DataSource 분리 필수
- Chunk 지향 처리 기본, chunk size 명시
- Job 멱등성 (재실행 = 중복 없음)
- faultTolerant skip/retry 정책 업무 요건에 맞게 설정

### spring-cache

- 로컬(Caffeine) vs 분산(Redis) 선택 먼저 확인
- `@Cacheable` key SpEL 표현식 명시
- TTL 반드시 설정
- null 캐시 방지 (`unless = "#result == null"` 또는 `disableCachingNullValues()`)
- `@CacheEvict` 타이밍 일관성 요건에 맞게 설정

### spring-observability

- Actuator 노출 `include` 방식으로 최소화 (`"*"` 금지)
- Micrometer 추상화 사용 (특정 벤더 직접 의존 금지)
- OTel tracing + sampling probability 명시
- Structured JSON logging
- Correlation ID MDC 전파 필터

### spring-async

- `SimpleAsyncTaskExecutor` 절대 사용 금지
- `@Async` 반환 타입: `void` 또는 `CompletableFuture<T>`만 허용
- `CompletableFuture` 예외 처리 (`exceptionally` / `handle`)
- `@Scheduled` 겹침 방지 (ShedLock 또는 `fixedDelay`)
- 스레드풀 core/max/queue 크기 명시

## 교훈

1. **§A meta-regression 확장이 안전망을 넓힌다.**
   retro #20 §A 는 스킬당 1개 규칙이었다. 이번 확장으로 21개 체크가 됐으며
   모두 PASS — 기존 문서 품질이 규칙 확장에서도 유지됨을 확인했다.
   스킬 수정 시 regression-22 를 먼저 돌리면 자기참조 위반을 조기 탐지할 수 있다.

2. **신규 스킬 v0.1은 기존 템플릿을 따르면 일관성이 보장된다.**
   anti-pattern Before 블록에 `// 금지` 주석을 붙이는 컨벤션 덕분에
   §A meta-regression 에서 허용 예외가 명확해진다.

3. **Phase 분리가 품질을 지킨다.**
   4개 신규 스킬을 이번 세션에 empirical 사이클까지 포함했다면
   각 스킬의 결함 수집이 얕아졌을 것. Phase B/C 이관으로 각 스킬이
   제대로 된 BASELINE → 2차 회전 사이클을 받게 된다.
