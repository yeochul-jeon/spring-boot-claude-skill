# 회고 #19 — @MockBean → @MockitoBean 마이그레이션 (Boot 3.4+ 대비)

날짜: 2026-04-22

---

## 사건 요약

retro #18 교훈 4 에서 follow-up 으로 기록된 항목을 해소한다.
`spring-testing` SKILL.md·references 가 `@MockBean`
(`org.springframework.boot.test.mock.mockito.MockBean`) 을 예시로 사용하고 있었다.

Spring Boot 3.4 / Spring Framework 6.2 부터 `@MockBean` 은 deprecated 되고,
`org.springframework.test.context.bean.override.mockito.MockitoBean` 으로 이관되었다.
본 프로젝트의 `spring-init` 은 런타임에 최신 Boot 버전을 조회하므로
새 세션에서 생성되는 코드는 즉시 Boot 3.4+·4.x 대상이다.
기존 예시 코드를 그대로 두면 deprecation warning 이 발생한다.

## 교훈

1. **"INFO" 항목은 방치하면 결함이 된다.** T8 는 retro #18 에서 "Boot 3.3.4 이므로 허용"
   으로 남겨졌으나, 프로젝트가 최신 Boot 를 따라가므로 즉시 해소하는 것이 옳았다.

2. **deprecated API 는 스킬 파일에서 즉시 교체해야 한다.**
   스킬은 "앞으로 생성될 코드의 기준"이므로, 예시 코드에 deprecated 심볼이 있으면
   해당 예시를 그대로 따라 쓴 코드에서 warning 이 쌓인다.
