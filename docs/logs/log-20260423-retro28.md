## 패치 상세

### 1. 버전 필터 인버트 — A7a / A8a / A9a / A10a

**문제**: `grep -v "gradle\|redis\|micrometer\|shedlock\|..."` 는 같은 라인에 Boot 버전이 있어도 숨김.
재현: `implementation("org.springframework.boot:spring-boot-starter:3.5.6") // gradle`

**수정**: positive 매칭으로 인버트 — `spring-boot` / `springframework.boot` 좌표를 직접 타겟.
```bash
UNSAFE=$(grep -rnE 'spring-boot.*[0-9]+\.[0-9]+\.[0-9]+|springframework\.boot.*[0-9]+\.[0-9]+\.[0-9]+' $SKILLS/X/ \
  | grep -v "latestVersion\|//\|#")
[ -n "$UNSAFE" ] && echo "$UNSAFE" && echo "FAIL" || echo "PASS"
```

외부 라이브러리(shedlock/caffeine/redis/micrometer/testcontainers) 버전은 자연 통과.

### 2. A9b — properties 비인용 wildcard 탐지

**문제**: `include.*"\*"\|include.*'\*'` 만 → `include=*` (properties 포맷) 미탐지.

**수정**: `include[ =:"'"'"']*\*` — YAML 쌍/단따옴표 + properties 비인용 세 변형 커버.
허용 맥락 필터에 `변형 모두\|세 변형` 추가 (SKILL.md 설명 주석).

### 3. A9d — `probability` 키 강제

**문제**: `sampling:` 단독만 있어도 PASS → `probability` 없이 통과 가능.

**수정**: `sampling\.probability\|probability:` — flat properties + YAML 중첩 모두 `probability` 확인.

### 4. A10b — `echo` → `=== [AS` 앵커

**문제**: `grep -v "echo"` 는 너무 넓음. `@Bean fun echoExecutor() = SimpleAsyncTaskExecutor()` 위반 숨김.

**수정**: `echo` 제외 → `=== \[AS` 앵커 (grep 명령 레이블 고유 구조).

### 5. A10c — 한국어 단어 → bash 주석 앵커

**문제**: `원시 스레드\|무한 스레드풀` 제외 넓음. 위반 라인 주석에 단어 포함 시 숨김.
재현: `Executors.newCachedThreadPool(); // 무한 스레드풀 임시 사용`

**수정**: `grep -Ev ':[0-9]+:[[:space:]]*# '` — grep -rn 출력 `file:N:# ` 앵커로 bash 주석만 정확히 제외.

## 교훈

1. **"포함" 기반 `grep -v` 필터는 구조적 취약점을 갖는다.**
   라이브러리명·설명 단어 단순 포함 제외는 미래의 위반 라인이 같은 단어를 언급할 때 숨김 위험. 타겟 심볼에 anchored pattern (bracket, `:N:` prefix, FQCN 접두) 사용이 더 견고.

2. **금지 규칙은 negative 필터보다 positive 타겟팅이 안전하다.**
   "전체 버전 → 허용 예외 제외" 패턴보다 "Boot 좌표만 매칭 → 실패 시 FAIL" 인버트가 외부 라이브러리 버전 false-positive 0건 + Boot 하드코딩 false-negative 0건을 동시 달성.

3. **존재-확인 규칙은 "키 이름"까지 내려가야 한다.**
   `sampling:` 같은 네임스페이스 레벨 매칭은 필요값(`probability`) 누락을 탐지 못함. 실제 설정 의미를 담는 leaf key 를 강제하는 것이 정확하다.

4. **Codex adversarial review 는 empirical 사이클의 정식 단계로 편입한다.**
   이번 리뷰에서 High 0 / Medium 8 — 상당수 FP/FN 가 정적 구조 분석으로만 드러났다. retro #20 이후 모든 회전 완료 시 codex 리뷰 1회 패스가 표준 절차로 정착.
