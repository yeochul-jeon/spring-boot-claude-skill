# 회고 #14 — spring-principles empirical 검증 (결함 I: VO 불변성 항목 누락)

날짜: 2026-04-22

---

## 사건 요약

`spring-principles` SKILL.md 의 구조 검증(케이스 4, 2026-04-21)은 통과했으나
실제 코드 생성 후 체크리스트 적용(empirical 검증)은 아직 수행되지 않았다.

서브에이전트로 "Order API + Money VO" 시나리오(b, 확장)를 생성하고
`spring-principles` SKILL.md 를 참조하지 않은 맨세션 조건에서
결함을 수집했다. 7개 결함이 모두 식별되었으며 그 중 하나(결함 I)는
SKILL.md 체크리스트에 항목이 아예 없었다.

## 발견된 결함

### 결함 I — VO 불변성 체크리스트 항목 누락

- 기존 SKILL.md: VO 도입 여부만 확인 (`Primitive Obsession은 VO로 추출`)
- 실제 코드: `Money.java` 에 `setAmount(int)` setter 가 존재 → 불변성 위반
- SKILL.md 어디에도 "VO 에 setter 있으면 위반"이 명시되지 않음
- `templates/value-object.md` 의 After 예시는 `record` (불변) 를 사용하지만
  체크리스트 항목에 반영되지 않아 코드 리뷰 시 탐지 누락 가능

### 결함 A~G (기존 항목, grep 패턴 부재)

아래 결함들은 SKILL.md 에 체크리스트 항목이 이미 존재하나
grep 기반 자동 탐지 수단이 없어 수동 리뷰에만 의존했다:

| 결함 | 내용 |
|------|------|
| A | `@Autowired` 필드 주입 (Controller·Service 각 1건) |
| B | Controller 가 `Order` Entity 를 직접 반환 (3개 메서드) |
| C | `@Transactional` 이 Service 가 아닌 Controller 에 배치 |
| D | 조회 메서드에 `@Transactional(readOnly = true)` 없음 |
| E | 상태 전이 로직(`confirm`, `cancel` 규칙)이 Service 에 있음 (Anemic Entity) |
| F | `Money.setAmount` 존재 (결함 I 와 동일 근원, grep 으로도 탐지 가능) |
| G | `private final` 필드 전무 |

## 교훈

1. **체크리스트 항목과 After 예시는 한 쌍이어야 한다.**
   `templates/value-object.md` 의 After 코드가 `record`(불변)를 사용하면서
   체크리스트에 "VO 는 setter 금지"가 없었다. 템플릿과 체크리스트가 분리되면
   새 세션이 After 코드를 못 보고 Before 패턴으로 회귀한다.

2. **grep 패턴이 없으면 수동 눈리뷰에만 의존하게 된다.**
   `@Autowired`, `@Transactional`(Controller), Entity 직접 반환은 grep 으로
   0.5 초 안에 탐지 가능하다. SKILL.md 에 grep 패턴을 고정하면
   이후 세션이 "체크리스트 실행"을 grep 커맨드로 구체화할 수 있다.

3. **맨세션 조건 baseline 수집이 스킬 품질을 측정하는 유일한 방법이다.**
   구조 검증(파일 존재 여부)은 쉽게 통과하지만, SKILL 을 참조하지 않은 세션이
   만들어낸 코드에서 결함이 탐지되어야 비로소 스킬이 제 역할을 하는지 알 수 있다.
