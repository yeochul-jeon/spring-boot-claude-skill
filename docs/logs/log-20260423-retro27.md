## grep 필터 보강 — A7d / A10b / A10c

### A7d `.chunk(` 패턴 수정

- v0 초안: `grep -rn "\.chunk("` — `.` 뒤에 바로 `chunk` 가 아닌 `.<T,R>chunk(` 형식을 미탐지
- 수정: `grep -rn "chunk("` (점 없이) — `>chunk(` 형식 포함

### A10b SimpleAsyncTaskExecutor echo 라인

- SKILL.md:144 `echo "=== [AS1] SimpleAsyncTaskExecutor / 비관리 스레드 ==="` — grep 명령 레이블 echo
- `# AS1` 필터가 `[AS1]` 를 커버하지 못함 → `echo` 를 exclusion 에 추가
- 판정: grep 명령 섹션 내 echo 레이블 — 허용 맥락

### A10c new Thread( bash 주석 라인

- SKILL.md:142 `# new Thread( — 원시 스레드 생성; Executors.newCachedThreadPool — 무한 스레드풀`
- `//` 필터가 bash `#` 주석을 커버하지 못함 → `원시 스레드\|무한 스레드풀` 를 exclusion 에 추가
- 판정: bash 주석 설명 맥락 — 허용 맥락

## 교훈

1. **grep -v 필터는 언어별 주석 스타일을 모두 커버해야 한다.**
   Java `//` 주석 필터가 bash `#` 주석을 커버하지 못한다. 다국어 환경의 문서(Java+bash 혼재)에서
   `//\|# ` 두 패턴을 함께 추가하는 습관이 필요하다.

2. **type witness 구문 `.<T,R>chunk(` 은 `\.chunk(` 패턴에 걸리지 않는다.**
   dot 이 꺾쇠 괄호 제네릭 앞에 오는 경우 `.chunk(` 가 아닌 `>chunk(` 형식으로 나타난다.
   범용적인 탐지에는 `chunk(` (dot 없이) 가 더 안전하다.

3. **§A meta-regression 은 스킬 생성 직후가 아니라 empirical 사이클 완료 후 한 번 더 실행해야 한다.**
   SKILL.md 가 empirical 검증 후 grep 패턴 추가 등으로 수정되면 자기참조 위반이 새로 생길 수 있다.
   Phase D 처럼 empirical 완료 후 §A 확장을 별도 단계로 수행하는 흐름이 올바르다.
