## 발견된 결함

### testing-domain 결함 (탐지 공백 3건)

| 결함 | 내용 | 분류 |
|------|------|------|
| T1 | `src/test/resources/application.yml` H2 인메모리 설정 — TestContainers 미사용 | 항목 존재, grep 없음 |
| T3 | `@DataJpaTest` + `@AutoConfigureTestDatabase(replace=NONE)` 누락 — H2 자동 치환 허용 | 항목 존재, grep 없음 |
| T4/T5 | TestContainers 의존성·`@Testcontainers` 완전 부재 | 항목 존재, grep 없음 |

세 결함 모두 SKILL.md 체크리스트에 이미 항목이 있었다.
grep 자동 검증 섹션이 없어 수동 확인에만 의존하는 구조였다.

#### T1 세부

- 기존 체크리스트: "통합 테스트에 H2 없음 — TestContainers로 실제 DB 엔진 사용"
- 생성된 코드: `src/test/resources/application.yml` 에 `jdbc:h2:mem:testdb` + `H2Dialect` 명시
- `@DataJpaTest` 기본은 내장 DB로 치환하므로, 맨세션 조건에서 H2 선택은 자연스러운 흐름
- grep `h2\|H2Dialect` (테스트 리소스 경로) 로 양성 탐지 가능

#### T3 세부

- 기존 체크리스트: "`@AutoConfigureTestDatabase(replace = NONE)` 적용 (`@DataJpaTest` 사용 시)"
- `@DataJpaTest` 기본 `replace = ANY` → 선언된 DataSource 무시하고 내장 DB로 치환
- TestContainers 를 쓰려면 반드시 `replace = NONE` 이 있어야 실제 컨테이너 DB 를 사용함
- T1 과 T3 는 세트 결함: H2 가 있으면 replace=NONE 이 없고, 반대도 성립

#### T4/T5 세부

- 기존 체크리스트: "`testImplementation("org.testcontainers:junit-jupiter")` 의존성 포함"
- `build.gradle.kts` 에 `testcontainers` 의존성 전무
- `@Testcontainers`, `MySQLContainer` 사용 없음
- grep `testcontainers` (build.gradle.kts) 및 `@Testcontainers` (테스트 소스) 로 탐지 가능

## 교훈

1. **체크리스트 항목 존재 ≠ 탐지 가능.** T1·T3·T4/T5 모두 항목이 있었으나
   grep 패턴 없이 수동 확인에만 의존했다. grep 섹션 추가로 이후 세션은 커맨드 한 줄로 탐지 가능.

2. **"금지" 항목만 있으면 "권장" 패턴이 빠진다.** T_Fixture 는 랜덤 금지 체크박스만 있었고,
   Fixtures/ObjectMother 사용이라는 양성(Positive) 항목이 없었다.
   금지 항목과 양성 항목을 함께 두어야 체크리스트가 완전해진다.

3. **맨세션 조건에서 H2 + @DataJpaTest 는 자연스러운 흐름이다.**
   TestContainers 설정은 복잡하므로, 스킬 없이는 H2 로 빠진다.
   이 패턴이 가장 자주 발생하는 결함이므로 grep 탐지가 가장 중요하다.

4. **`@MockBean` → `@MockitoBean` 마이그레이션 필요 (Boot 3.4+).**
   SKILL.md 예시 코드가 `@MockBean` 을 사용하고 있으며, Spring Boot 3.4+ 에서 deprecated.
   현재 sandox 는 Boot 3.3.4 로 결함 아니나, `@MockitoBean` 으로 업데이트 권장 — 별도 follow-up.
