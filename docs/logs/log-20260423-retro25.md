- v0.1 O5 grep: `sampling.probability\|tracing` — YAML 다중 레벨 중첩(`sampling:` / `probability:`) 에서 `sampling.probability` 한 줄 탐지 불가
- `tracing:` 키 하나만으로 false-PASS 반환 — sampling 미설정 시 오탐 위험
- 분류: **탐지 공백** — BASELINE 실제 코드에서는 `probability: 1.0` 설정됨(latent)

- 분류: **탐지 공백** (latent)
- 수정: `include.*'\*'\|include=\*` 변형 추가

## 교훈

1. **YAML 다중 레벨 중첩은 단일 문자열 grep 을 우회한다.**
   `management.tracing.sampling.probability` 는 YAML 에서 네 개의 중첩 키로 분리된다.
   `sampling.probability` 한 줄 패턴은 properties 포맷에서만 동작한다.
   YAML 검증 시 개별 키(`sampling:`, `probability:`)를 분리 탐지하고 수동 확인을 병행해야 한다.

2. **인용 변형(단일/큰따옴표, properties 포맷)을 모두 커버해야 한다.**
   금지 패턴(wildcard `*`)은 YAML 큰따옴표, 작은따옴표, properties `=` 세 가지 변형이 있다.
   grep 에서 하나라도 빠지면 개발자가 변형 표기로 우회 가능.

3. **잘 알려진 패턴은 BASELINE PASS 가능하다 — 탐지 공백 수집이 핵심이다.**
   spring-observability(Micrometer, OTel, MDC)는 업계 표준 패턴이라 SKILL 없이도
   모범 구현이 나왔다. SKILL 의 가치는 grep 패턴이 미래 변형에서도 false-PASS 를
   내지 않도록 보강하는 데 있다.
