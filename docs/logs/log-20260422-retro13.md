# 회고 #13 — Boot 4.x 최소 Gradle 버전 오기재 (결함 H)

날짜: 2026-04-22

---

## 사건 요약

회고 #12 반영 후 `test-retro11/order-api/build.gradle.kts` 에
`id("io.spring.dependency-management") version "1.1.7"` 을 추가하고
`./gradlew build --no-daemon` 를 실행했다. 그러나 빌드가 다시 실패했다:

```
> Failed to apply plugin 'org.springframework.boot'.
   > Spring Boot plugin requires Gradle 8.x (8.14 or later) or 9.x.
     The current version is Gradle 8.13
BUILD FAILED in 7s
```

wrapper 는 회고 #12 재검증 로그에서 `gradle wrapper --gradle-version 8.13` 으로
생성되었고, 참조 문서도 "Boot 4.x → Gradle 8.4+" 로 기재되어 있어 올바른 버전으로
인식되었다. 그러나 Spring Boot 4.0.5 플러그인은 실제로 8.14 이상을 요구한다.

---

## 발견된 결함

### 결함 H — Boot 4.x 최소 Gradle 버전 오기재

- 기존 문서: "Boot 4.x → Gradle 8.4+" (4개 파일, 5개 위치)
- 실제 요구사항 (empirical): Boot 4.0.5 플러그인 오류 메시지에서 확인
  `Spring Boot plugin requires Gradle 8.x (8.14 or later) or 9.x`
- 영향: fallback scaffold 및 wrapper 갱신 시 8.4~8.13 을 지정해도 문서상 허용이나
  실제 빌드 실패. 회고 #11~12 의 "빌드 가능성" 판정이 이 결함으로 계속 미성립.

## 교훈

1. **정적 호환표는 Boot 패치 릴리스 경계에서 stale 된다.**
   "Boot 4.x" 와 같이 major.x 로 범위를 잡은 표는 패치 릴리스(4.0.0 → 4.0.5 등)에서
   내부 요구사항이 조용히 상향될 수 있다. 테이블보다 **빌드 실패 로그의 오류 메시지**가
   더 신뢰 가능한 근거다.

2. **wrapper 생성 직후 `./gradlew help` 로 플러그인 적용 성공을 확인해야 한다.**
   wrapper 생성 자체는 성공해도 bootVersion 플러그인이 Gradle 버전 요건을 충족하지
   않으면 모든 gradlew 명령이 실패한다. wrapper 생성 → `./gradlew help` 성공 →
   이후 단계 진행을 표준 순서로 고정한다.

3. **`Spring Boot plugin requires Gradle ...` 오류 메시지를 파싱해 재지정한다.**
   이 메시지는 요구 Gradle 버전을 명시적으로 포함한다. 문서상 버전보다 이 메시지를
   우선해 `gradle wrapper --gradle-version <요구값>` 을 재실행하면 된다.
