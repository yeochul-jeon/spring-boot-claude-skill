# 회고 #12 — fetch 스크립트 BSD 호환성 + dependency-management 버전 누락

날짜: 2026-04-22

---

## 사건 요약

회고 #11 반영 후 `tests/regression-11-fallback-buildability.md` §2 테스트 A 를
재실행했다. `generate-project.sh` 에 `exit 56` 을 삽입해 fallback 경로를 강제
유발했으며, Claude 는 §2-c 수동 scaffold 로 프로젝트를 생성했다.

결함 A~F 체크리스트는 명목상 전부 통과했으나, 재검증 과정에서 2건의 신규 결함이
드러났고 `./gradlew build` 가 실행되지 않아 "빌드 가능성" 판정이 성립하지 않았다.

---

## 발견된 결함

### 결함 A 재발 — BSD sed alternation 비호환

- `fetch-latest-versions.sh` 가 `bootVersion = "4.0.5.RELEASE"` 를 반환 (정제 실패).
- 원인: `sed 's/\.\(RELEASE\|...\)$//'` 는 GNU sed 전용 문법.
  macOS BSD sed 에서 `\|` 는 alternation 이 아닌 리터럴 `|` 로 처리되어 매칭 실패.
- Claude 가 `.RELEASE` 를 수동으로 제거해 `4.0.5` 를 `build.gradle.kts` 에 기입.
  결과물은 올바르지만 **설계상의 안전망이 작동하지 않는 상태**였다.
- 수정: `sed -E 's/\.(RELEASE|BUILD-SNAPSHOT|RC[0-9]*|M[0-9]*)$//'` 로 변경.
  `sed -E` 는 BSD/GNU 모두 확장 정규식 플래그로 동작.

### 결함 G 신규 — dependency-management 플러그인 버전 누락

- `references/gradle-conventions.md §1` 예시가 `id("io.spring.dependency-management")` 버전 없이 기재.
- `io.spring.dependency-management` 는 Spring Boot Gradle plugin 에 번들되지 않은
  **독립 외부 플러그인**. 버전을 명시하지 않으면 Gradle plugin portal 조회가 실패.
- 재검증 로그에서 `gradle wrapper --gradle-version 8.13` 이 다음 오류로 실패:
  ```
  Plugin [id: 'io.spring.dependency-management'] was not found in any of the following sources
  ```
- Claude 가 임시 프로젝트로 wrapper 만 생성해 복사하는 우회 조치를 취했으나 근본 해결 아님.
  `./gradlew build` 실행 시 동일 오류가 반복된다.
- 수정: 참조 문서와 SKILL.md §2-c 예시에 `version "1.1.7"` 추가.

### 검증 공백 — `./gradlew build` 미실행

- 체크리스트 전 항목 통과 후 `./gradlew build` 를 실행하지 않았다.
- 회고 #11 의 핵심 목표 "fallback 산출물의 빌드 가능성" 이 실증되지 않은 채로 마무리.
- 수정: `regression-11-fallback-buildability.md` 에 `./gradlew build --no-daemon` 필수
  실행 커맨드 블록 + stdout 마지막 5줄 첨부 의무 추가.

## 교훈

1. **스크립트 정제 로직은 실행 환경(BSD vs GNU)에서 독립 검증해야 한다.**
   `sed`, `tr`, `awk` 등 POSIX 유틸리티는 macOS BSD / Linux GNU 간 동작이 다르다.
   특히 alternation(`\|` vs `|`) 과 character class 처리에 차이가 있다.
   스크립트 작성 후 macOS 에서 직접 실행해 동작을 확인한다.

2. **체크리스트 통과 ≠ 실빌드 통과 — 빌드 커맨드 실행 증거 없이는 판정 금지.**
   "빌드 가능성" 을 목표로 삼은 회고에서 `./gradlew build` 를 실행하지 않고 완료로
   처리한 것은 목표 자체를 검증하지 않은 것이다.
   빌드 관련 회고의 판정 기준은 항상 실 빌드 커맨드 실행 + 출력 첨부로 고정한다.

3. **참조 문서 예시는 실제로 동작하는 코드여야 한다.**
   `gradle-conventions.md` 의 플러그인 예시가 버전을 누락한 채로 존재했고,
   Claude 가 그 예시를 그대로 복사해 결함이 전파되었다.
   참조 문서의 코드 블록은 복사-붙여넣기 즉시 동작해야 한다는 원칙을 적용한다.
